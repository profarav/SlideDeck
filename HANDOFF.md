# Klimt Slide Engine — Handoff

**What it is:** An internal tool for Klimt & Design salespeople. Given a prospect description (or their website URL), it retrieves the most relevant case study slides from the 631-slide portfolio deck and can assemble a full ordered pitch deck. Results are exportable as a PDF.

**Repo:** `github.com/profarav/SlideDeck`  
**Live URL:** Deployed on Vercel (check Vercel dashboard for current URL)  
**Stack:** Python / FastAPI backend · Vanilla JS SPA frontend · Anthropic Claude API

---

## How It Works

```
Prospect description / URL
        ↓
   profiler.py          — Claude maps prospect to canonical tags
   (industries, service_categories, visual_style, search_query)
        ↓
   retriever.py         — 5-level fallback filter + keyword pre-rank + Claude final rank
   (returns slides with "why this slide" explanations)
        ↓
   api.py               — FastAPI serves results + PDF export
        ↓
   static/index.html    — SPA renders slide cards with image previews
```

### Five-level retrieval fallback
1. Industry **+** service_category (exact match)
2. Service_category only (right work type beats right industry)
3. Industry only
4. General Agency slides (1–24)
5. Entire library

### Slide index
- `vision_index.json` — 631 entries, one per slide, generated by Claude Vision (haiku-4-5)
- Each entry: `slide_number, page_id, filename, client, industry, visual_style, service_type, service_category, content, text, why`
- `slide_index.json` — fallback (117 range-based entries from the Excel sheet, used only if vision_index.json is missing)

## File Map

| File | Purpose |
|------|---------|
| `api.py` | FastAPI app. All HTTP endpoints. |
| `profiler.py` | Turns raw prospect text into structured search persona via Claude. |
| `retriever.py` | Multi-stage retrieval: filter → keyword pre-rank → Claude ranking. |
| `deck_builder.py` | Assembles full pitch deck (intro + case studies + closing). |
| `tags.py` | Canonical tag lists + range-based slide taxonomy (authoritative ground truth). |
| `vision_ingest.py` | One-time (or resumable) script: sends each PNG to Claude Vision to build `vision_index.json`. |
| `ingest.py` | Reads Master Sheet.xlsx → `slide_index.json` (fallback index, rarely needed). |
| `sync.py` | Google Slides API sync — detects new/moved slides using stable `page_id`, updates `vision_index.json`. |
| `cli.py` | Terminal test interface for retrieval pipeline. |
| `static/index.html` | Full frontend SPA (single file, no build step). |
| `slides_png/` | 631 PNG exports of every slide. Excluded from Vercel (too large), served via GitHub raw CDN. |
| `.github/workflows/sync-slides.yml` | GitHub Action: runs `sync.py` nightly at 2 AM UTC. |

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | Serves the SPA |
| `POST` | `/generate-deck` | Case study finder. Returns flat list of ranked slides. |
| `POST` | `/build-deck` | Full deck builder. Returns sections: Agency Overview → Case Studies → Next Steps. |
| `POST` | `/export-pdf` | Takes ordered slide numbers, returns compiled PDF download. |
| `GET` | `/health` | `{"status": "ok"}` |

**Request body for `/generate-deck` and `/build-deck`:**
```json
{
  "description": "Series B fintech building embedded payments...",
  "url": "https://prospect.com",   // optional, fetched + scraped if provided
  "n_results": 4
}
```

## Slide Index — Rebuilding

The index is pre-built and committed to the repo. Only rebuild if something is broken or you want to re-analyze all slides.

```bash
pip install -r requirements-sync.txt   # adds Pillow (already in requirements.txt)
python vision_ingest.py --slides ./slides_png
# Resumable — safe to re-run, skips already-processed slides
# ~$8 for 631 slides at $0.013/slide (claude-haiku-4-5)
# ~30-40 minutes at 10 parallel workers
```

## Syncing New Slides (Ongoing Maintenance)

When the designer adds slides to Google Slides, the sync runs automatically every night. It can also be triggered manually.

**How the sync works:**
1. Fetches current slide list from Google Slides API
2. Matches by `page_id` (permanent Google object ID — stable even when slides are inserted mid-deck)
3. Downloads thumbnails for new slides only
4. Runs Claude Vision on new slides only
5. Updates `slide_number` for any slides that moved position
6. Commits updated `vision_index.json` back to GitHub → Vercel auto-redeploys

**Manual trigger:**
- GitHub → Actions tab → "Sync Slides" → Run workflow
- Or locally: `python sync.py --presentation-id <SLIDES_ID>`

**One-time Google Cloud setup (already done or needs to be done once):**
1. Google Cloud Console → New Project → Enable Google Slides API
2. IAM & Admin → Service Accounts → Create → Download JSON key
3. Share the Google Slides deck with the service account email (view access)
4. Add to GitHub Secrets:
   - `ANTHROPIC_API_KEY`
   - `GOOGLE_SLIDES_ID` (the ID from the Slides URL)
   - `GOOGLE_SERVICE_ACCOUNT_JSON` (full contents of the JSON key file)

## Vercel Deployment

The repo is connected to Vercel. Every push to `main` auto-deploys.

**Bundle size:** `slides_png/` (360 MB) is excluded via `.vercelignore`. Images are served directly from GitHub raw CDN (`raw.githubusercontent.com/profarav/SlideDeck/main/slides_png/`).

**Required Vercel env var:** `ANTHROPIC_API_KEY`

**PDF export on Vercel:** Fetches slide PNGs from GitHub raw at export time (parallel, ~3-5 seconds for a full deck). Works fine on paid Vercel plans. Hobby tier has a 10s function timeout which may be tight for large decks.

## Known Limitations / Planned Improvements

- **Vision at ranking time** — Currently ranks by text descriptions. Passing actual slide images to Claude during the final ranking step would significantly improve quality (planned, ~$0.05–0.15/query).
- **Multi-query expansion** — Generate 3 search angles from the persona, merge candidate pools before ranking (planned).
- **Canonical tag validation** — Profiler output isn't validated against canonical lists; a hallucinated tag silently breaks filters (planned fix: fuzzy-match against `CANONICAL_*` lists after profiler returns).
- **PDF export timeout** — On Vercel Hobby tier, exporting large decks (10+ slides) may hit the 10s timeout. Upgrade to Pro or resize images before compiling.
- **slides_png not on Vercel** — Vision-at-ranking-time won't work on Vercel without object storage (Vercel Blob or S3) since the PNG folder is excluded from the bundle.
